I have created a pets adoption service website as static pages to demonstrate how it would look and potentially function, but now it is time to make it smart with back end functionality so we can maintain the site with admin logins.

I want to be able to accept applications for adoption, accept applications for fostering, accept message requests on the contact us page and redirect the message to our email address, I want to create a profile for an animal, edit the profile, archive it, and delete it.  

In the adoptionlist I would like to activate the filter functionality, and add a sex filter, plus also show the sex on the preview of the animal.

I will need a superadmin and regular admin access so we can access back end functionality.  
I will also need the ability to export CSV files for adoption applications, foster applications, contact messages, list of animals, list of news articles, and anything else you can identify.  I would also like an admin page with stats about the page, how many animals adopted, how many site visits, what animals have how many views, esc

I would like all pictures of animals to be square, so if the image is not square, I want to give the uploader the option of zooming and cropping (and choose the midpoint), or to put bars top and bottom or left and right with the colour #f5f5f5.


You will see on each webpage there is a header menu, a footer, and a script to allow for paw prints to randomly appear on the screen.  I would like them to be taken away from the HTML pages and put them into one script/py so I can edit it once for global change.

from django.db import models
from django.utils.text import slugify

class AnimalProfile(models.Model):
    STATUS_CHOICES = [
        ('AVAILABLE', 'Available Now'),
        ('SOON', 'Available Soon'),
        ('PENDING', 'Pending Adoption'),
        ('ADOPTED', 'Adopted'),
    ]
    
    animal_name = models.CharField(max_length=100)
    slug = models.SlugField(unique=True, blank=True)
    species = models.CharField(max_length=50) # Dog, Cat, etc.
    breed = models.CharField(max_length=100)
    sex = models.CharField(max_length=10)
    adoption_fee = models.DecimalField(max_digits=10, decimal_places=2)
    status = models.CharField(max_length=20, choices=STATUS_CHOICES, default='AVAILABLE')
    date_entered = models.DateTimeField(auto_now_add=True)
    primary_image = models.ImageField(upload_to='animal_photos/')
    description = models.TextField()

    def save(self, *args, **kwargs):
        if not self.slug:
            self.slug = slugify(self.animal_name)
        super().save(*args, **kwargs)

class AnimalImage(models.Model):
    """The Gallery Model for extra photos"""
    animal = models.ForeignKey(AnimalProfile, related_name='gallery', on_delete=models.CASCADE)
    image = models.ImageField(upload_to='animal_photos/gallery/')

class BaseApplication(models.Model):
    """Shared fields for everyone"""
    first_name = models.CharField(max_length=100)
    last_name = models.CharField(max_length=100)
    email = models.EmailField(unique=True)
    mobile = models.CharField(max_length=20)
    date_submitted = models.DateTimeField(auto_now_add=True)

class AdoptionApplication(BaseApplication):
    """Specific to Adoption"""

class FosterApplication(BaseApplication):

Project Overview

Migrate the static HTML/CSS rescue site into a Django-powered web application. The system will support:

    Animal profile management (CRUD + archive)

    Adoption and foster applications

    Contact form with email relay

    News/story publishing with scheduling

    Administrative dashboards with stats

    CSV export of key datasets

Authentication & Roles

    Admins: Use Django’s default admin system.

        Regular Admins: Can manage animal profiles and news articles.

        Superadmins: Same as admins, plus ability to delete other admin accounts.

    Adopters/Fosterers: No login required to submit applications.

Application Workflow

    Applications trigger email notifications to pawsamomentandrescue@gmail.com.

    Confirmation emails sent to adopters/fosterers upon submission.

    Applications are not editable by users after submission.

    Applications stored in DB for reporting/export.

Animal Profile Management

    CRUD operations: Create, Edit, Archive (soft delete), Delete (permanent).

    Archived animals remain in DB but hidden from public listings.

    Adoption status updated manually by admins.

Filtering & Search

    Active filters:

        Name/Description (text search)

        Species (multi-select dropdown)

        Size (multi-select dropdown)

        Sex (multi-select dropdown)

    Implementation: Filters should be server-side (Django queries) for scalability, security, and consistency.

Admin Dashboard

    Key statistics:

        Adoptions per month

        Foster placements

        Site visits

        Animal profile views

    Tracking: Use Django middleware + database logging for site visits/views. (Google Analytics optional later.)

CSV Export

    Exports downloadable from admin panel.

    Export all records (no filter restrictions).

    Supported exports:

        Adoption applications

        Foster applications

        Contact messages

        Animal profiles

        News articles

Image Handling

    Uploads limited to 1000x1000 pixels, JPG format to conserve storage.

    Cropping handled in-browser (JavaScript) for user experience, with server-side validation/resizing using Python/Pillow.

    Options for cropping or adding bars (#f5f5f5 background).

News/Stories

    Expired news auto-hidden after newsexpiry.

    Admins can schedule posts for future publication.

Suggested Additions

    Email Integration: Configure SMTP or third-party service (e.g., SendGrid) for notifications and contact form relay.

    Payment Integration: Plan for Stripe/PayPal integration; donations logged in DB.

    Audit Logs: Track admin actions (create/edit/delete) for accountability.

    Accessibility: Ensure paw prints and image cropping do not interfere with screen readers.

    Scalability: Consider Docker setup and deployment options (Heroku, AWS, etc.).

Technical Requirements

    Language: Python 3.10+

    Framework: Django 4.2+

    Database: PostgreSQL recommended for scalability.

    Slug System: Required for Animals and News (/animals/max-labrador/).

    Folder Structure:

        /static/ → UI assets (logos, paw prints)

        /media/ → User uploads (animal photos, yard photos)

        /templates/ → HTML templates
